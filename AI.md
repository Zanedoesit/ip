## AI tools used for

- A-CodeQuality: Assisted in spotting redundant code and code that did not meet standards
- A-GUI: Assisted with troubles I faced during version compatibility in IntelliJ when trying to compile my code
- A-MoreOOP: Provided different options for implementation and structure which I took into consideration
- A-Gradle: Assisted with debugging
- A-JUnit: Verified tests
- A-Jar: Assisted with debugging for runtime errors and version incorrect naming
- General Debugging: Helped diagnose runtime errors and build issues encountered

## Observations
- AI is surprisingly helpful with helping of build issues and quick for identifying issues with code.
- AI is also very fast at generating skeleton code that might need further refining or prompting which saves time.
- However, these suggestions still need manual verification and AI still lacks the contextual understanding of the 
project like we have.
- Some logical faults from occasional misunderstanding of requirements.
